# Prompt リファクタチェックリスト (prompt-refactor-checklist.md)

## 目的

- リファクタ作業で必ず確認する観点を定義し、不要な追加や語弊を防ぐ。

## 実行前チェック

- [ ] 変更は failure-log のどのエントリに対応するか明示した
- [ ] 追加ではなく、まずは言い換え・統合・移設で対応できるか検討した
- [ ] どの層（常設 / skill / agent）へ配置するか決めた

## 実行後チェック

- [ ] 重複している規則は残っていないか
- [ ] 同じ原則を別表現で複数書いていないか
- [ ] 長い行動規範が常設 prompt に残っていないか（第2層へ移すべき）
- [ ] 例外条件が本文に埋もれていないか（必要なら専用の skill に移す）
- [ ] 第3層（agent）固有の権限や禁止事項は agent prompt にあるか
- [ ] 変更の理由と影響範囲を prompt-failure-log.md に記録した

## 品質ゲート

- 変更は原則として "削除・統合・移設" を優先し、明確な理由がある場合のみ新規追加を許可する。
- 変更後は軽い差分レビューを行い、少なくとも1人のレビュー承認を得ること。
