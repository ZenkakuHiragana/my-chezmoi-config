# 義務語彙の棚卸し

## 目的

prompt、command、agent、skill の指示文では、要求の強さを日本語の正規マーカーで表す。
正規マーカーは、人間と linter が同じ義務レベルを判定するための基準である。
RFC 2119 風の英字マーカーは分類の参考名として扱い、prompt 本文の義務表現には使わない。

## 義務レベル

- 必須: 要求する挙動。満たさなければ契約違反である。
  - 「〜する」
  - 「必ず〜する」
  - 「〜しなければならない」
- 禁止: 禁止する挙動。実行してはならない。
  - 「〜してはならない」
  - 「禁止する」
- 推奨: 原則として採用する挙動。外すときは理由を持つ。
  - 「〜すべきである」
  - 「推奨する」
- 非推奨: 原則として避ける挙動。採用するときは理由を持つ。
  - 「〜すべきでない」
  - 「避ける」
- 任意: 任意の挙動。実行しなくても契約違反ではない。
  - 「〜してよい」
  - 「任意とする」

## 曖昧表現

次の語句は義務レベルを決めない。要求する挙動には使わない。

- 「向いている」
- 「合う」
- 「場合だけ」
- 「使える場合だけ」
- 「可能なら」
- 「必要なら」
- 「できれば」
- 「するとよい」
- 「望ましい」

## 書き換え規則

- 要求する挙動は必須の正規マーカーで書く。
- 禁止する挙動は禁止の正規マーカーで書く。
- 推奨に留める挙動は推奨または非推奨の正規マーカーで書く。
- 任意の挙動は任意の正規マーカーで書く。
- 義務レベルを決められない文は、要求、禁止、推奨、任意のどれかに分類してから書く。
- どの義務レベルにも属さない曖昧表現は、指示が無効化されたものとして扱う。
- 曖昧表現を見つけたら、義務レベルへ割り当てるか削除する。
