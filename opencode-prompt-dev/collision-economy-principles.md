# 衝突経済の設計原理

## 概要

『根源課題の地図と検証中心の設計原理』(fundamental-problem-map.md) は、AI 委任の失敗を還元不能な5つの根源課題 P1〜P5 に分類し、証明書搬送型の作業規律を中心投資と定めた。本報告はその後継となる概念整理であり、課題体系を設計軸と診断軸の2軸に分離して見取り図を更新する。

先行文書との関係を先に固定する。P1〜P5 の定義、`problem_classes` の schema、certificate 規律、実装課題 B1〜B6 は変更しない。変更するのは P1〜P5 の役割である。P1〜P5 は失敗を記帳する診断台帳として使い、介入の採否判断には本報告の設計法則を使う。識別子は失敗記録の時系列集計が依存する schema であり、改名や再定義は分布の時系列を切断するため行わない。

本報告は人間向けの概念整理に限る。実装課題と計画は扱わず、先行文書の第2部を引き続き正本とする。

### 結論

- 課題体系は2軸に分離する。設計軸は衝突経済(介入をどう選ぶか)、診断軸は P1〜P5 台帳(失敗をどう記帳するか)である。1本の木に畳むと、診断分類が設計基準のふりをする。
- 設計軸の中心法則: 成果物は、産出時か消費時に回避不能な衝突が予定されているときに限り品質装置として機能する。衝突が予定されていない成果物は、実行時圧力の下で儀式化する。
- 産出時の衝突は捏造コスト反転(正直に作る方が偽装より安い状態)で設計し、消費時の衝突は反証手段を持つ消費者への送達で設計する。
- P1/P4 の残余は命題的な穴と認識的な穴に分かれ、処方が異なる。前者は責務と照合、後者はプローブでしか届かない。
- `problem_classes` の分類自体が P4 の制約を受け、強いモデルは失敗の見かけのクラスを変えうる。停止規則の発火には分類の検証を前提とする。
- 帳簿は消費財(タスク出力)と資本財(計器、schema、fixtures、地図文書)で分ける。資本財は将来の衝突単価の低下で測り、held-out 回帰評価への接続を投資条件とする。
- 衝突コスト階段の左端は territory の写像との衝突にすぎない。忠実度は高価な衝突だけが供給し、fixtures パイプラインがそれを左端の計器へ輸送する。
- 検査の脱相関は、文脈の脱相関と事前分布の脱相関を区別する。後者はモデル族、機械照合、人間、territory との接触でしか買えない。
- 要約の連鎖はフレーム汚染の伝播経路である。要約境界に一次資料ポインタ、reframing、fresh reader による再構成を置く。

## 用語

先行文書の用語(certificate、resolver、rubric、消費者文脈、意図ミラー、衝突コスト階段、`problem_classes`、統治予算、sunset ablation)はそのまま使う。本報告で追加する語は次のとおり。

- 衝突: 成果物、主張、前提が、それを反証しうる対象(機械照合、コンパイル、実行、消費者、現実)と突き合わされる事象。
- 衝突の予定: 衝突が回避不能な形で工程に組み込まれていること。resolver の照合、merge を止める quiz、意図の持ち主への意図ミラー提示がこれにあたる。
- 捏造コスト反転: 成果物を正直に作るコストが、偽装して作るコストを下回るように設計された状態。
- 命題的な穴: 真偽を持つが未確認の前提。証明責務として列挙できる。
- 認識的な穴: 見れば判別できるが言語化されていない基準。命題として列挙できない。
- 忠実度: 検査結果が territory の実態と一致する度合い。
- フレーム汚染: 前段の要約や問題設定に混入した誤りが、後段で挑戦されずに増幅される現象。

## 二軸構造

先行文書は P1〜P5 を課題の分類と介入の対応先の両方に使っていた。この兼任が見取り図を窮屈にする。「P2 に対応する介入」という言い方は介入の宛先を示すだけで、その介入が効くかどうかを決めない。readiness 閂は P1 に対応する介入だったが、blind A/B で gate-rate を改善しなかった(2026-06-25、baseline = commit 02229fd)。certificate は P2 に対応する介入であり、作成時点で誤りを主張不能にする機序を持つ。宛先の分類は両者を区別できず、区別するのは次節の設計法則である。

したがって2軸に分ける。

```text
設計軸(介入をどう選ぶか) — 衝突経済
  ├─ 産出時の衝突: 捏造コスト反転(certificate / resolver)
  ├─ 消費時の衝突: 消費者による反証(意図ミラー / quiz / prototype)
  ├─ 衝突の複利化: 高価な衝突を fixtures として左端の計器へ還流する
  ├─ 衝突路の脱相関: 文脈の脱相関と事前分布の脱相関を区別する
  └─ 衝突能力の保全: 文脈物流と注意予算(モデルと人間の両方)

診断軸(失敗をどう記帳するか) — P1〜P5 台帳【凍結】
  ├─ problem_classes の付与と分布集計
  └─ 停止規則(分類の検証を前提に発火)
```

介入は設計軸で選び、失敗は診断軸で記帳する。P2 が全課題の管理コストを決める貨幣単位であるという先行文書の分析は変わらない。変わるのは、その貨幣で何を買うかを決める基準の置き場である。

## 中心法則: 衝突の予定

### 観測の対比

現行システムで効いた装置と効かなかった装置を並べると、1つの軸で割れる。

| 装置 | 衝突の予定 | 観測結果 |
| --- | --- | --- |
| certificate + resolver | 産出時。引用とパスが機械照合される | 出典欄を埋める義務が `inktype.lua` の参照を強制し、誤った主張が作成時点で成立しなくなる(失敗記録 20260628-1706 の反実仮想)。blind A/B 最小評価パックの smoke と fresh validation を critical failure 0 で通過 |
| 意図ミラー / quiz / prototype | 消費時。意図、理解、好みを唯一持つ人間が安く反証できる | unknown knowns(質問では取れない基準)に届く観測が『A Field Guide to Fable』に整理されている |
| 失敗ログ → fixtures | 事後。高価な衝突が安い計器の較正に使われる | 稼働中。fuck.jsonl 終盤の顛末整理から `review-response` skill への結実が追跡できる |
| readiness 閂 | なし。同じ文脈から接地なしに生成できる | blind A/B で gate-rate 改善ゼロ |
| watchdog の状態自己申告 | なし。申告は省略も偽装もできる | 状態 YAML を出さないモデルに対して監視が成立しなかった(『コンテキスト管理と compaction の調査報告』) |
| prose の契約義務、concern 分割義務 | なし | 実行時圧力の下で破られる(triage 20260628-1546 の C003、fuck.jsonl の契約不履行) |

この対比から中心法則を立てる。

成果物は、産出時か消費時に回避不能な衝突が予定されているときに限り品質装置として機能する。衝突が予定されていない成果物は、実行時圧力の下で儀式化する。義務の記述が prose に存在するかどうかは、この判定を変えない。

先行文書は「作成の時点で多くの誤りが主張不能になる」を certificate の効能として記述した。本報告はこれを certificate の性質ではなく、全介入に適用する採否基準へ一般化する。readiness 閂の null 結果、watchdog の不成立、C003 は、独立の失敗ではなく、この基準を満たさない装置の3標本として読める。

### 産出時の衝突と捏造コスト反転

産出時の衝突は、成果物を作る行為そのものが territory との接触を強制するときに成立する。certificate の出典欄は、埋めるためにファイルを開くことを要求し、開いた時点で誤った主張が死ぬ。resolver の抜き取り照合は、偽装の発覚確率を上げて偽装の期待コストを引き上げる。両者が揃うと、正直に作る方が偽装より安いという捏造コスト反転が成立する。

この構造は経済学のシグナリング理論(Spence 1973)と同型である。信号が信頼できるのは、偽の信号を出すコストが正直な信号を出すコストを上回るときに限られる。また、儀式化の機序は Goodhart の法則(指標が目標になると指標は指標でなくなる)の実例である。prose の義務は「義務を満たしたように見える出力」という指標を最適化させ、捏造コスト反転を持つ成果物だけがこの最適化を無効にする。

### 消費時の衝突と消費者の反証手段

産出時に接地を強制できない成果物でも、反証手段を持つ消費者に必ず届くなら衝突は予定されている。意図ミラーの消費者は意図の持ち主であり、解釈のずれを一目で反証できる。quiz の消費者は自分の理解度を答案の正誤で反証される人間である。prototype の消費者は「見れば分かる」基準の持ち主である。いずれも、成果物の正しさを消費者側の私有情報と突き合わせる設計になっている。

逆に、反証手段を持たない消費者への送達は衝突にならない。下流 skill が構造的に読み込むだけの record、確認なしに受領される報告は、消費者が存在しても衝突が存在しない。成果物の価値は送達ではなく反証可能な受領で決まる。

### 介入評価の問い

介入とレビューの採否では、次を順に問う。第0問が本報告の追加であり、残りは先行文書の枠組みの言い換えである。

0. この成果物の衝突はいつ、誰と予定されているか。正直に作る方が偽装より安いか。
1. これは何を、どの表現から、どの表現へ変換するのか。
2. その変換で、どの検証コストが下がるのか。
3. その検証は、誰の消費者文脈で行われるのか。
4. その失敗は、次回どの低コスト衝突へ蓄積されるのか。

第0問に答えられない介入は、宛先の課題分類が何であれ、null 結果の系譜に加わる見込みが高い。

## 残余の2分類: 命題的な穴と認識的な穴

P1(意図の伝達不能性)と P4(認識の自己申告不能性)は解消できず管理だけができる。この管理対象は内部構造を持ち、2種に分かれる。

命題的な穴は、真偽を持つが未確認の前提である。「`ID_*` は pass-local である」「この API は再入可能である」がこれにあたる。証明責務として列挙でき、certificate、出典照合、default-deny で扱える。

認識的な穴は、見れば判別できるが言語化されていない基準である。視覚設計の好み、文体の許容範囲、「これは違う」としか言えない要件がこれにあたる。命題ではないため、責務としての列挙も質問による抽出も原理的に届かない。届くのは prototype、reference、実物比較というプローブであり、消費時の衝突の特殊形である。『A Field Guide to Fable』の unknowns 4分類のうち Unknown Knowns がこの穴を指す。

2分類の取り違えには実害がある。認識的な穴に主張表を当てると、表は埋まるが基準は露出しない。命題的な穴にプローブを当てると、確認1回で済む事実に試作コストを払う。P1/P4 の管理装置を選ぶ手前で、穴の種類を判別する。

## 会計上の注意

### 分類の P4 汚染

`problem_classes` の付与は失敗発生後の観測に基づく判断であり、判断者がモデルであれ人間であれ、真因への直接アクセスを持たない。分類という行為自体が P4 の制約下にある。

さらに、強いモデルは失敗の見かけのクラスを変えうるという仮説がある。SNS 観察コーパスの統合分析(fable-5-thoughts.md)は、Fable 級モデルの諸観察を「文脈全体への一貫性圧が強い」という単一性質で説明し、欠けた前提を文脈整合的な値で無音に補完する挙動(coherent gap-filling)を導いた。この仮説が正しいなら、文脈に情報が存在したのに上書きされた失敗(P3/P4 起源)や、検査が高くて無音の補完を捕まえ損ねた失敗(P2 起源)が、「意図が伝わっていなかった」という P1 の顔で記帳される。

帰結として、主因分布の P1/P4 集中は「工学的余地が尽きた」証拠と「モデルが失敗のクラスを偽装するのが上手くなった」証拠を区別しない。停止規則(P1/P4 集中でプロンプト体系の改良を停止する)をこの分布だけで発火させると、偽装の勝利宣言になりうる。発火の前提として、P1 判定に観測条件を課す: 意図は実際に取得不能だったのか、それとも文脈に存在したのに上書きされたのか。後者は P1 ではない。fable-5-thoughts.md にはこの仮説を判別する blind eval 設計3件(フレーム固着、信念分離、修正非対称)が既にあり、分類の検証手段として転用できる。

停止規則の意味も限定する。停止するのは prose 規則への追加投資であり、プローブ経路、計器、相互作用プロトコルへの投資ではない。P1/P4 の管理装置はそれ自体が改良対象であり続ける。

### 消費財と資本財の帳簿分離

このリポジトリの産物は2種に分かれ、会計が異なる。

消費財はタスクの出力であり、そのタスクの成功で測る。資本財は計器(resolver、check_vocabulary.py、blind-eval ハーネス)、schema、fixtures、地図文書であり、将来の衝突単価をどれだけ下げるかで測る。資本財の価値は産出時点では確定せず、held-out の回帰評価に接続されて初めて測定可能になる。この接続を持たない資本財の増設は、局所過適合した規則の堆積と同じ経路をたどる。

先行文書の装置はこの帳簿で読み直せる。sunset ablation は資本財の減価償却テストであり、モデル世代の進歩で価値を失った規則を除却する。「所与の品質水準を達成できる最弱のモデル」という最終指標は、資本ストックの総量測定である。失敗記録は fixtures の在庫、すなわち仕掛品である。

## 忠実度の還流

衝突コスト階段(意図ミラー < certificate 照合 < resolver < 分離レビュー < コンパイル < 実機プローブ < 利用者の手戻り)はコストの序列であり、忠実度の序列はおおむね逆向きに走る。左端の安い衝突は territory そのものではなく、territory の写像(仕様書、wiki、certificate の引用元)と衝突している。写像が古い、または誤っているとき、安い衝突は偽陰性を返す。放置された wiki をエージェントが確信をもって出力する問題(『根拠収集と知識検索の調査報告』)がこの形である。

忠実度を供給するのは右端の高価な衝突だけである。コンパイル、実機プローブ、利用者の手戻りは territory と直接衝突し、写像の誤りごと露出させる。失敗ログ → fixtures → 計器の較正というパイプラインの正体は、右端で1回だけ支払った忠実度を、左端の安い計器へ輸送する装置である。

帰結: 右端の衝突をゼロへ最適化すると、左端の較正が止まり、安い衝突の忠実度が沈黙のうちに劣化する。右端の衝突は削減対象であると同時に、定期的に支払う較正コストである。矛盾の発見を階段の左へ移すという先行文書の原則は維持した上で、左端の検査結果を信頼できる状態に保つ費用が右端で発生し続けることを会計に含める。

## 脱相関の2種

多重検査の価値は見逃しの独立性で決まり、買うべきは層の枚数ではなく脱相関である。この脱相関には2種があり、コストと効果が異なる。

文脈の脱相関は、fresh context の検証者、フレーミングの変更、文脈スライスの変更で買える。安いが、届く範囲は文脈由来の相関までである。同族モデルは fresh context でも訓練由来の事前分布を共有する。

事前分布の脱相関は、モデル族を跨ぐ、機械照合へ落とす、人間が見る、territory と直接接触させる、のいずれかでしか買えない。失敗記録 20260628-1706 の見逃しは事前分布相関の実例である。「単一使用の定数は局所化せよ」は訓練分布上の一般ベストプラクティスであり、ドメイン正本(`inktype.lua` の row schema)と衝突していた。この種の見逃しは、同族モデルのレビュアーを何枚重ねても、fresh context にしても、相関したまま残る。

検査設計では、どちらの脱相関を買っているかを明示する。文脈の脱相関で足りるのは生産者文脈への過適合(盲点継承、anchoring)であり、事前分布の脱相関が要るのは一般常識とドメイン真実が衝突する領域である。

## 要約連鎖とフレーム汚染

複数モデル、複数セッションにまたがる分析ワークフローには、単一セッションにはない伝播経路がある。SNS 観察コーパスの統合分析が立てた仮説によれば、Fable 級モデルは確立した主張への圧力には反論する一方、自信をもって提示されたフレームには従順に乗る。前段のモデルが書いた要約は、後段のモデルにとって自信をもって提示されたフレームであり、要約に混入した誤りは挑戦されずに一貫的に増幅される。

要約は表現変換の一種であり、変換のたびに何が保存され何が落ちるかが変わる。要約の連鎖は、この損失を検査なしに累積させる構造を持つ。対策は3つで、いずれも衝突の予定を要約境界に置く形をとる。

- 一次資料ポインタの保持。要約には出典パスを残し、主張の照合先を切らない。certificate の原理の文書間適用にあたる。
- 要約境界での reframing。問題設定そのものを疑う問い(この文脈全体を踏まえて、本当に問うべきだった問いを立てよ)を、局所修正の累積時、同じ失敗の再発時、分析の段の切り替わりで発火させる。
- fresh reader による再構成。要約の消費者に、要約ではなく一次資料から芯を再構成させ、要約との差分を検査する。消費者文脈の原理を文書系譜へ適用したものである。

## この枠組み自体の限界と反証条件

中心法則は少数の観測からの遡及的な統一であり、統一の美しさは正しさの根拠にならない。バラバラの観察をエレガントな単一物語に統合するのは一貫性圧の強い生成器の得意技であり、その注意は本報告自身に適用される。したがって反証条件を明示する。

- resolver 照合を持たない certificate 義務(捏造検出なし)が、照合ありと同等の主張品質改善を blind A/B で示すなら、捏造コスト反転の寄与は過大評価であり、義務の存在自体が効いていることになる。certificate schema の評価パック(certificate-blind-ab-v0.md)はこの比較に拡張できる。
- 衝突が予定された装置(quiz gate、deviations log)が、prose 上の予定だけでは実地で発火しないことが観測され続けるなら、「予定」は prose では買えない。その場合、中心法則の適用範囲は機械監査可能な衝突(resolver の実行記録、ファイルとセクションの存在検査)に縮小する。
- runtime 強制点が opencode / codex / claude を横断して存在しないという構造的天井は、本報告でも消えない。衝突の予定そのものが prose に置かれる限り、同じ強制ギャップを受ける。逃げ道は、衝突事象が事後に機械観測可能であること(照合ログ、成果物の存在)に限られる。

本報告の装置はすべて床を上げる。衝突の予定は誤りの露出を安く早くするが、開いた設計判断の質、トレードオフの重み付け、要件の解釈は、依然として相互作用と人間の判断に残る。
