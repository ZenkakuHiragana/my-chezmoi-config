# Prompt 失敗ログ (prompt-failure-log.md)

## 目的

- プロンプト運用で発生した「失敗」「意図しない出力」「誤った決定」を記録し、再発防止とリファクタリングの素材にする。

## 運用ルール

- 失敗が確認されたらすぐにこのファイルへ項目を追記する。
- 記入は簡潔に。担当者が後で詳細を補える形式にすること。
- 各エントリは少なくとも 1) 日付 2) 対象 agent/skill 3) 入力の要約 4) 期待した出力 5) 発生した出力 6) 根本原因の仮説 7) 推奨対応（移設/言い換え/新規追加） を含む。

## エントリテンプレート

---

日付: YYYY-MM-DD
対象: (agent 名 / skill 名 / ワークフロー)
プロンプト参照: (ファイル名 と 行/セクション など任意)
入力（要約）: (ユーザー入力やトリガの要約)
期待される挙動: (期待した出力/判断を簡潔に)
実際の挙動: (実際に得られた出力・挙動)
既存規則で防げたか: (はい / いいえ / 不明) と簡単な理由
根本原因（仮説）: (箇条書き)
推奨対応案: (例: 常設規則へ統合 / skillへ移設 / 新規ルール候補を作成)
優先度: (高/中/低)
ステータス: (open / in_progress / resolved)

---

## 例

日付: 2026-04-01
対象: general-assistant
プロンプト参照: dot_config/opencode/AGENTS.md#Language Policy
入力（要約）: ユーザが多言語で短く挨拶した（中国語を含む）
期待される挙動: 日本語での返答。中国語を出力しない。
実際の挙動: 中国語の挨拶を一部含む応答が返った
既存規則で防げたか: いいえ — 規則自体は存在したが優先順位と表現が曖昧だった
根本原因（仮説）: 規則の表現が長く分散しており、agent は具体的な処理順序を決められなかった
推奨対応案: prompt-principles.md の「intent gate」項目に明記して再配置
優先度: 中
ステータス: open

---

## 注意

- このファイルは記録が目的であり、ここで直接プロンプト本文を編集する必要はない。編集は prompt-refactor ワークフローに従うこと。
