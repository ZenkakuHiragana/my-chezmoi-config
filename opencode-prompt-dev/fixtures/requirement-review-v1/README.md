# 要件review fixture

## 目的

`requirement-review`が、明示要求と契約の有限な対応だけを検査し、一般論から条項を増やさないことを確認する。

## 対象

- 明示要求の欠落
- 根拠のない契約条項
- 実装が所有する詳細の契約への複製
- 問題のない契約への過検出

## 非対象

- code reviewと日本語文章reviewの品質
- ユーザーが明示していない意図の推測
- 全種類の仕様書と業務領域
- model間の性能比較

## 適用限界

小さい合成例なので、実契約の長さ、複数資料の衝突、公開仕様の誤りは再現しない。成功はskillの完全性を証明せず、4つの既知失敗を区別できることだけを示す。

## 期待される効果

- 明示要求の取りこぼしと無認可追加を、契約本文の完全化と分離する。
- code、test、runtimeが所有する詳細を契約へ転記する再発を抑える。
- clean契約へ質問や条項を追加する過検出を可視化する。

## 実行

`scenarios.md`の各scenarioを別のfresh実行者へ渡し、`scoring.md`で採点する。実行者へ採点基準を渡してはならない。
