# レビュー結果の形式

```markdown
# 日本語本文review結果

## 確認範囲

- 対象:
- 想定読者:
- 利用目的:
- 観点:

## 確認済みの問題

### ISSUE-001

- 観点: STRUCTURE / GRAMMAR / STYLE / TYPO
- 重要度: 高 / 中 / 低 / None
- 対象箇所:
- 読者への破綻:
- 根拠:
- 確認方法:

## 未確認事項

- <None または確認できなかった事項と理由>

## 判定

- 合格 / 不合格
```

- 指定された全観点を`確認範囲`へ列挙する。
- 問題が0件なら`確認済みの問題`へ`None`と書き、合格とする。
- 問題が1件以上なら不合格とする。
- `TYPO`以外は、本文の引用と読者に起きる具体的な破綻を示す。
- `TYPO`は誤表記と一意な正表記を示す。
- 未確認事項は修正要求にしない。
